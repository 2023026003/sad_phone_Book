# PRD.md — 전화번호 관리 시스템

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 전화번호 관리 시스템 (Phone Book Manager) |
| 목적 | 전화번호를 웹에서 쉽게 입력, 수정, 삭제할 수 있는 간단한 CRUD 시스템 |
| 대상 사용자 | 로그인 없이 접속하는 일반 사용자 |
| 기술 스택 | Next.js (프론트엔드/API) + Supabase (DB) + Vercel (배포) |

---

## 2. 기능 요구사항

### 2.1 핵심 기능 (MVP)

| 기능 | 설명 |
|------|------|
| 전화번호 조회 | 저장된 전체 연락처 목록을 표시 |
| 전화번호 추가 | 이름 + 전화번호를 입력하여 새 연락처 등록 |
| 전화번호 수정 | 기존 연락처의 이름 또는 전화번호를 변경 |
| 전화번호 삭제 | 특정 연락처를 목록에서 제거 |

### 2.2 비기능 요구사항

- 로그인/인증 불필요 (공개 접근)
- 페이지 새로고침 없이 동작 (SPA 방식)
- 모바일 반응형 UI
- 전화번호 형식 기본 유효성 검사

## 4. 데이터 모델

### Supabase 테이블: `contacts`

| 컬럼명 | 타입 | 제약 | 설명 |
|--------|------|------|------|
| id | uuid | PK, default gen_random_uuid() | 고유 식별자 |
| name | text | NOT NULL | 연락처 이름 |
| phone | text | NOT NULL | 전화번호 |
| created_at | timestamp | default now() | 생성 시각 |
| updated_at | timestamp | default now() | 수정 시각 |

---

## 5. API 설계

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | /api/contacts | 전체 연락처 조회 |
| POST | /api/contacts | 연락처 추가 |
| PUT | /api/contacts/[id] | 연락처 수정 |
| DELETE | /api/contacts/[id] | 연락처 삭제 |

---

## 6. 기술 스택 상세

| 구분 | 기술 | 역할 |
|------|------|------|
| 프론트엔드 | Next.js 14 (App Router) | UI 렌더링 |
| 스타일 | Tailwind CSS | 반응형 스타일링 |
| 백엔드 | Next.js API Routes | REST API |
| 데이터베이스 | Supabase (PostgreSQL) | 데이터 저장 |
| 배포 | Vercel | 호스팅 |
| DB 클라이언트 | @supabase/supabase-js | DB 연동 |

---

## 7. 환경 변수

```env
NEXT_PUBLIC_SUPABASE_URL=<Supabase 프로젝트 URL>

```

---

## 8. 제약사항 및 가정

- 인증 없이 누구나 CRUD 가능 (Supabase RLS 비활성화 또는 public 정책 적용)
- 전화번호 중복 체크는 미구현 (MVP 범위 외)
- 페이지네이션 미구현 (초기 버전)
- 검색은 클라이언트 사이드 필터링으로 구현
